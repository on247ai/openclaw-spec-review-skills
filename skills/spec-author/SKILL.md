# Spec Author

Dùng skill này khi cần viết spec trước khi implement.

## Mục tiêu
Khóa yêu cầu để implementation không lệch bài toán.

## Cách làm
1. Tóm tắt feature.
2. Viết user stories / actors.
3. Liệt kê functional requirements.
4. Liệt kê non-functional requirements nếu có.
5. Mô tả main flows / error flows.
6. Bổ sung edge cases.
7. Chốt acceptance criteria.
8. Map acceptance criteria sang test cases.

## Template
Đọc `assets/SPEC_TEMPLATE.md`.

## Review
Đọc `references/spec-review.md` khi cần review nhiều vòng.

## Output
- `docs/specs/SPEC-xxx-name.md`
